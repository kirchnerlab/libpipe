libpipe
libpipe is a lightweight C++ pipelining library. Which allows pipelining of filters. The library provides a straightforward and simple interface that allows easy integration into new and existing projects.
libpipe has so far been applied ... . The library itself, however, is in no way limited to this application scenario: it can easily be used kinds of problems which needs pipelining of algorithms.
libpipe development has been conducted at the Steen & Steen lab.
Development, Documentation and Testing
The libpipe code is considered stable; it is actively maintained at the libpipe github site. If you happen to run into trouble, please do not hesitate to contact Marc or to make use of the github issue tracking system.
We make use of the Doxygen documentation system to provide source documentation.
libpipe comes with a set of fully automated regression tests, that check all classed for correct behavior.
Download and Licensing (MIT)
libpipe is available under a permissive MIT license that allows anyone to use, modify and distribute libpipe. Please consider contributing your improvements.
Please note: libpipe makes heavy use of the boost libraries, in particular threads, ptree and shared_ptr. Development was carried out under GNU gcc 4.6.1 with boost 1.46.0, on an Ubuntu Linux system.

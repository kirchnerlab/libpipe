Detailed Description

A filter that supports runtime configuration. Filters are non-copyable composite objects that combine a Manager and an Algorithm object. Note that the Filter destructor is not virtual, that Filter is hence not designed to serve as a base class and that extension of filter functionalities should be implemented by extending its constituents, i.e. Algorithm and Manager.

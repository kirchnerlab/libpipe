Detailed Description

Generic parameter collection class. The Parameters class is a (somewhat) generic collection of key/value pairs intended for use as a parameter object. The implementation is based on an underlying variant that can take any type but has currently been limited to 

      std::string, std::vector<std::string> >

Using Parameters instances is straightforward, however there are a few best practices:

  • in most cases a Parameters instance will be part of e.g. a Filter or an algorithm class/object. In this case, the containing class should provide get functionality to a properly instantiated Parameters object, that has been initialized with the correct required and optional parameter names (i.e. keys) as well as set functionality for the complete Parameters instance. 
         // SomeObject constructor:
     SomeObject::SomeObject(...) : ...
     {
         ...
         std::vector<std::string> req, opt;
         req.push_back("delta");
         opt.push_back("solution");
         params_ = Parameters(req, opt);
         ...
     }
  • in such cases, the client code should not attempt to construct a parameter object from scratch but instead copy the existing object: 
        Parameters p = someObject.getParameters();
    p.set("delta", "0.773");
    p.set("solution", "42");
    someObject.setParameters(p);
  • before using the values in a Parameters object, the client should validate the contents to see if the required keys are present: 
         ...
     bool isGood = p.validate();
     ...
     if (!isGood) {
         throw UnleashMayhem("Oops.");
     }
     ...

See the tests for more examples.

     ...
     Parameters p(req, opt);

     ...

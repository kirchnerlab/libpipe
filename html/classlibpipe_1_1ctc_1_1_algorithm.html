bool needUpdate ( ) const

Returns a flag that signifies if the Algorithm needs an update, irrespective of its modification time. The function essentially tests it the modification time of the algorithm is equal to Algorithm::MAX_TIME, signifying that the algorithm needs to be run and that its modification time value should be disregarded. Managers must check needUpdate prior to making use of the information delivered by getMTime. There are two major application scenarios that make use of the flag:

  • When algorithms are instantiated for the first time, their creation date need not coincide with the execution sequence (this is especially true for filter setups with branches). Hence, their modification time is set to Algorithm::MAX_TIME to guarantee their execution on the first call to update.
  • When the precursors are deleted the modification time gets set to Algorithm::MIN_TIME to guarantee that the algorithm does not update himself.
  • Managers in filters that act as sources (i.e. do not have any predecessor) cannot determine the need for execution by comparing Algorithm::getMTime() against their predecessor. In the event that their internal state changes, the manager needs to call this->getAlgorithm()->setMTime(Algorithm::MAX_TIME), thus enforcing an update of the source and propagation along the filter chain.
    Returns:
    Boolean flag that indicates the need for a call to the algorithm's update function.

virtual void update ( Request &  req) [pure virtual]

Runs the algorithm and updates the output data, if the Request Type is Request::Update. If the request Type is Request::DELETE after calling the update function the modification time is fixed to Algorithm::MIN_TIME. Users must overload this method (this is where the beef goes).

Parameters:
[in,out]reqThe request object (useful for logging/tracing purposes). Also useful to get the type of request (Update of Delete), so that the input can be deleted.
